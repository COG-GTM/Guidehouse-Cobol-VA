The proposal

What we build, what changes, what we add

A factory whose deliverable is the reconciliation evidence. The code exists to produce it.

  • 1
    Reconciliation engine
    Row accounting (in == loaded + rejected), dollar control totals to the cent, per-document debit==credit balance, mapping-coverage %.
  • 2
    Explicit, versioned target contracts
    Every conversion targets a Momentum import contract that tests assert against — never "legacy → vibes."
  • 3
    Momentum import simulator
    Re-reads the emitted file as Momentum would and re-asserts balance — a load rehearsal, not just a format check.
  • 4
    CI reconciliation gate
    The conversion exits non-zero when a batch isn't load-ready, so "did the money survive?" is a build status.
  • 5
    Agent orchestration + Knowledge Fabric
    An orchestrator fans out child sessions across interfaces; a growing knowledge base makes each wave faster and more accurate.

How this differs from the current AIE / A0–A8 plan as we understand it.
